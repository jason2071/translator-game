User Manual

คู่มือ RPGMaker Translator

แอปแปลเกม RPG / visual-novel — ด้วยมือหรือ AI. รองรับ RPGMaker MV/MZ, Ren'Py, TyranoScript, KiriKiri, Godot, และ Assassin's Creed. คู่มือนี้ไล่ ทุกเมนู ทุก feature ตามลำดับการตั้งค่า.

เปิดเกม→ตั้ง provider + key→context + glossary→Run→ตรวจ/แก้→Export

ภาพรวม

โครงหน้าจอ 2 สถานะ + 4 modal.

  • ยังไม่เปิดโปรเจกต์ → ทั้งหน้าคือ Import screen.
  • เปิดแล้ว → Sidebar (ซ้าย) + พื้นที่หลัก (ขวา): Update banner (ถ้ามี) → TranslateBar → Grid.
  • 4 modal เปิดทับ: Glossary, Glossary lint, AI providers & settings, Translation errors. ปิดด้วยปุ่ม X, Esc, หรือคลิกพื้นมืดนอก modal.
  • Theme สลับ dark/light (ไอคอนพระอาทิตย์/จันทร์) — default dark, จำค่าไว้.
Quit guard: ปิดหน้าต่างระหว่างแปล → เตือน Quit while translating?. batch ที่เสร็จ save แล้ว, ที่เหลือหยุด.

สีสถานะ (ใช้ทั้งแอป)

Untranslated Failed Draft Translated Reviewed Locked

เริ่มเร็ว — first run 11 ขั้น

ลำดับจริงตั้งแต่เปิดเกมถึง Export. รายละเอียดแต่ละเมนูอยู่หัวข้อถัดไป.

  1. เปิดโปรเจกต์ — Import → Choose game folder… → auto-detect engine → ดูการ์ด engine/ไฟล์. (คนเคยใช้คลิก Recent แทน — ข้าม detect, ใช้ภาษาเดิม)
  2. เลือกภาษา — From (default Auto) → To (default Thai) → Open project (สกัดข้อความ)
  3. ตั้ง provider + key — Sidebar → Settings → เลือก tab, ใส่ Model, Base URL (ถ้า local), API key + Save, ปรับ Temperature/Batch/etc. → Test connection
  4. เลือก Run provider — dropdown ใน TranslateBar (ต้องมี key ก่อน Run ยกเว้น Local)
  5. Game context + Era — Sidebar → Glossary → panel บนสุด: เลือก Era, เลือก glossary provider, AI draft (ถ้าอยาก), แก้ context
  6. สร้าง glossary — modal เดิม: Auto-suggest / AI suggest → Translate empty → แก้ → Add selected
  7. เลือก scope + Run — Sidebar เลือก All files หรือไฟล์เดียว → เลือก Overwrite existing → Run. (Apply TM ก่อนเติมของซ้ำฟรี)
  8. ตรวจ + แก้ — Grid: filter, แก้คำ (Ctrl/Cmd+Enter = save + ไปแถวถัดไป), ดู Preview + คำเตือน, ตั้ง status
  9. จัดการ error — เปิด Errors ดูเหตุผล → Retry failed
  10. Lint — Sidebar → Lint → แก้คำที่ไม่ตรง glossary
  11. Export — Sidebar → Export → game (backup อัตโนมัติ; ติ๊ก Embed Thai font สำหรับ RPGMaker MV/MZ)

1เปิด / สร้างโปรเจกต์

Import screen — โผล่เมื่อยังไม่มีโปรเจกต์เปิด.

เลือกโฟลเดอร์

  • Choose game folder… → picker "Select game folder". ระหว่าง detect ขึ้น Detecting…
  • ไม่เจอ engine → error แดง No supported game engine detected in this folder.

การ์ด Detection (เมื่อ detect สำเร็จ)

แถวแสดง
Engineชื่อ engine (ตัวหนา)
Data filesจำนวนไฟล์ข้อมูล
Data dirpath โฟลเดอร์ข้อมูล (monospace)
warningsคำเตือน (เช่น ระบบภาษาในเกม) — ไม่บล็อก

เลือกภาษา

  • From — Auto, Japanese, English, Chinese, Korean (default Auto = ให้โมเดลเดา)
  • To — Thai, English, Japanese, Chinese, Korean (default Thai)
  • Open project (ขึ้น Extracting…) → สกัดข้อความครั้งแรก + เปิดโปรเจกต์

Recent projects (ถ้ามีประวัติ)

  • หัวข้อ Recent + ลิงก์ Clear (ล้างทั้งหมด)
  • แต่ละแถว: ชื่อโฟลเดอร์, engine badge, คู่ภาษา source → target, progress bar + done/total, เวลา, ปุ่ม X ลบ
  • เปิดล้มเหลว → Couldn't open — the folder may have moved or been deleted. + Remove
เปิดจาก Recent ข้าม detect + เลือกภาษา — ใช้ภาษาที่ save ไว้ และไม่สกัดข้อความใหม่.

2TranslateBar — แถบเครื่องมือบน

ซ้าย = config ต่อโปรเจกต์, ขวา = ปุ่มสั่งรัน, ล่าง = แถบสถานะ.

ซ้าย — config

  • สลับภาษา — source → target (เปลี่ยนแล้ว update ทันที; disabled ระหว่าง Run)
  • Run provider dropdown — เลือก provider ที่ Run ใช้. ลำดับ: Local (Ollama / LM Studio), OpenAI, Claude (Anthropic), Gemini (Google), OpenRouter

ขวา — ปุ่มสั่ง

  • Target: {file} — scope ที่ Run จะทำ (ไฟล์เดียวหรือ All files). เปลี่ยนที่ Sidebar
  • Overwrite existing — ติ๊กเพื่อแปลทับที่มีคำแล้ว. ปิด = ข้าม
  • Run — แปล unit ที่เป็น Untranslated + Failed ของ scope. ระหว่างรันปุ่มกลายเป็น Cancel
  • Re-translate matches (N) — โผล่เมื่อมี search filter — แปลทับทุก unit ที่ match (ยืนยันก่อน)
  • Retry failed (N) — โผล่เมื่อมี Failed — แปลซ้ำเฉพาะที่ล้มเหลว
  • Errors (N) — เปิด modal Translation errors

แถบสถานะ (ล่าง)

  • progress ของ Run + Glossary คู่กัน
  • สรุปหลัง Run: Done — N translated, N reused, M failed (เลข failed คลิกได้ → กระโดดไป filter Failed)
  • error ระดับคำสั่ง (ไม่มี key / ไม่มีโปรเจกต์) แสดงแดงตรงนี้; per-unit ไปที่ Errors modal
คิวงาน: Run กับ Glossary แชร์ engine เดียว ทำ ทีละงาน — สั่งอันที่สองจะ queue อัตโนมัติ ไม่ถูกปฏิเสธ.

3Sidebar — แถบข้าง

dashboard + filter + ปุ่มลงมือ + export.

Dashboard

  • วงแหวน % = (total − untranslated) / total
  • 3 ตัวเลข: {n} done (Translated + Reviewed), {n} todo (Untranslated), {n} failed (คลิก → filter Failed)

Filter — Status

ปุ่ม (จุดสี + ชื่อ + จำนวน): All, Todo, Failed, Draft, Translated, Reviewed, Locked. คลิก → กรอง grid ตาม status.

Filter — Files

All files + แถวต่อไฟล์ (มีจำนวน unit, hover เห็น path). คลิกไฟล์ = ตั้งทั้ง grid และ Run scope (Target:).

ปุ่มลงมือ (ล่าง)

ปุ่มทำอะไร
Apply TMเติมจาก translation memory + คำซ้ำ — รายงาน "Filled N from memory". ทำก่อน Run เพื่อประหยัด
Glossaryเปิด modal Glossary
Lintเปิด modal Glossary lint
Settingsเปิด modal AI providers & settings
Export → gamebackup + inject กลับเกม (ดูหัวข้อ 10)
Embed Thai fontเฉพาะ RPGMaker MV/MZ + sidebar ขยาย. default on เมื่อ target = Thai. ฝัง font ไทยเข้าเกมกันตัวสี่เหลี่ยม (tofu)
Close project (X)กลับ Import screen

4ตารางข้อความ (Grid)

รายการ unit ทั้งหมด — แก้คำ + ตั้ง status ที่นี่.

แถบ search/filter (บน grid)

  • ช่อง search — placeholder Search source / translation… (Enter). กด Enter เพื่อค้น (ค้นทั้ง source + translation)
  • Untranslated only checkbox
  • N shown = จำนวนที่ match (filter ไฟล์/สถานะอยู่ที่ Sidebar)
  • ว่าง → No units match the current filter. + Reset filters

คอลัมน์

Source · context | Translation | Status

แต่ละแถว

  • Source — ข้อความต้นทาง + meta: ชื่อไฟล์, context/ผู้พูด, kind (Dialogue, Choice, Message…)
  • Translation — textarea โตอัตโนมัติ. แก้แล้ว blur = save. Ctrl/Cmd+Enter = save + ไปแถวถัดไป; Enter เฉยๆ = ขึ้นบรรทัดใหม่. แก้ครั้งแรกของ Untranslated/Failed → เป็น Draft
  • ⚠ codes differ — inline control code ไม่ตรง source (รู้ไวยากรณ์ code ตาม engine: RPGMaker \C[..], Ren'Py [..]/{..}, ฯลฯ)
  • ⚠ line too long — เฉพาะ kind กล่อง (Dialogue, ScrollText, Choice, Message) เมื่อเปิด width guard
  • Preview / Hide preview — kind กล่อง: จำลองกล่องข้อความ RPGMaker (สี \C[n], token \N[k] ฯลฯ, ชื่อผู้พูด)
  • ปุ่ม re-translate ต่อแถว — แปลบรรทัดนี้ใหม่ด้วย Run provider (ทับ). disabled ระหว่าง Run
  • Status dropdown — Untranslated, Failed, Draft, Translated, Reviewed, Locked

5Settings — AI providers & settings

ตั้งค่าแต่ละ provider อิสระ. ไม่ได้เลือกว่าอันไหนใช้ Run/glossary — Run เลือกที่ TranslateBar, glossary เลือกที่ Game context panel.

Provider tabs

Local (Ollama / LM Studio), OpenAI, Claude (Anthropic), Gemini (Google), OpenRouter — เลือก tab เพื่อแก้ config ของ provider นั้น.

ฟิลด์ต่อ provider

  • Model — พิมพ์ model id + ปุ่ม Refresh (list โมเดลที่มี) → dropdown — pick one of N installed —
  • Base URL — placeholder (default). Local default http://localhost:11434/v1
  • Temperature — 0–2, step 0.1, default 0.3
  • API key — ทุก provider ยกเว้น Local. password + Save / Clear. เก็บใน OS keychain, ไม่เก็บใน browser

6Game context + Era

อยู่บนสุดของ modal Glossary. per-project.

  • Game context (this project) — textarea lore/setting: ยุค, ตัวละคร, ความสัมพันธ์, โทน, กฎโลก. ป้อนให้ AI ทุก Run. save อัตโนมัติหลังหยุดพิมพ์
  • AI draft — ร่าง brief จากข้อความในเกมเอง (มี context อยู่แล้วยืนยันก่อนทับ)
  • Glossary provider dropdown — provider สำหรับ glossary + context help. แยกจาก Run provider

Era dropdown — seed register/สรรพนาม

ใส่สรรพนาม/ระดับภาษาให้ prompt ทุก Run (ประกอบกับ context อิสระ):

7Glossary — คลังคำ

คำที่ป้อนให้ AI + ใช้ lint ความสม่ำเสมอ (proper noun, ค่าสถานะ, ชื่อไอเทม).

Suggest panel

  • Auto-suggest from game — สแกน heuristic หา candidate
  • AI suggest — ใช้ glossary provider ขุด proper noun ที่ heuristic พลาด
  • มี candidate แล้ว: header N candidates · N filled + action:
    • Translate empty (N) — แปลทุกคำที่ว่าง/failed (queue หลัง Run)
    • Re-translate all (N) — แปลทับทุกคำ
    • Add selected — เขียนคำที่ติ๊กเข้า glossary
    • Cancel — ทิ้ง candidate set
  • แต่ละแถว: checkbox, term (hover เห็น kind ×count), ช่อง translation, ⚠/✓, ปุ่ม retry ต่อคำ
candidate + translation + สถานะ เก็บต่อเกม — ปิด modal/แอปแล้วยังอยู่. "Running in background — safe to close this dialog."

เพิ่มเอง + ตาราง

  • ฟอร์ม: Source term, Translation, Note (optional), Aa (case-sensitive), Add
  • ตาราง: Term / Translation / Note / ลบ — แก้ inline, save on blur

8Lint — ตรวจความสม่ำเสมอ

รันอัตโนมัติเมื่อเปิด modal.

  • สะอาด → ✓ No glossary inconsistencies found.
  • มีปัญหา → N translation(s) miss their glossary wording. + รายการ {file} — term {X} should map to {Y} (คลิกชื่อไฟล์ → กระโดดไปไฟล์นั้น + ปิด modal)

ตรวจอะไร: คำแปลที่ใช้คำสำหรับ term ไม่ตรงกับที่ glossary กำหนด (proper noun / ชื่อไอเทม / ค่าสถานะ).

9Errors — ที่แปลล้มเหลว

เปิดจากปุ่ม Errors (N) ใน TranslateBar.

  • ไม่มี → ✓ No translation errors.
  • มี → N unit(s) failed. Click one to jump to its file, then use 'Retry failed'.
  • แต่ละแถว: source, ชื่อไฟล์, เหตุผลที่ล้มเหลว. คลิก → filter ไปบรรทัด failed ของไฟล์นั้น + ปิด modal
  • แปลซ้ำที่ปุ่ม Retry failed (N) ใน TranslateBar

10Export → เกม

เขียนคำแปลกลับเข้าเกม. ปุ่ม Export → game ใน Sidebar.

  • backup เสมอ ก่อนแตะไฟล์
  • ส่ง option Embed Thai font เฉพาะ RPGMaker MV/MZ เมื่อติ๊ก
  • สำเร็จ → note ตาม engine (เช่น Ren'Py เขียน tl/<lang>/) หรือ Exported N units → M files + "(backup saved)"
ไฟล์เกม read-only จนกว่า Export. state ทั้งหมดอยู่ใน sidecar <game>\.rpgtl\. Export idempotent — export ซ้ำได้ผลเดิม (snapshot ไฟล์ต้นฉบับไว้).

ข้อควรระวัง — dependency

  • ต้องเปิดโปรเจกต์ก่อน ทำอะไรก็ได้ ยกเว้น config แบบ Settings
  • ต้องตั้ง API key + เลือก Run provider ก่อน Run (Local/Ollama ไม่ต้อง key)
  • glossary/context AI ใช้ glossary provider — แยกจาก Run provider คนละตัว
  • Run + Glossary job ทำทีละงาน — อันที่สอง queue
  • Embed Thai font มีเฉพาะ RPGMaker MV/MZ
  • Overwrite existing ปิด = ข้ามที่แปลแล้ว
  • Apply TM ก่อน Run เติมของซ้ำ/ตรงฟรี — ประหยัด token
  • Export ทำ backup เสมอ; ไฟล์เกมไม่ถูกแตะจนกว่า Export
รันนานแล้วช้าลง? ส่วนใหญ่มาจาก provider (Ollama GPU ร้อน / API rate-limit) — นอกการควบคุมของแอป. ลอง batch เล็กลงหรือโมเดลเบากว่า.
คู่มือนี้สร้างจากการสำรวจ UI จริงของแอป (ทุก view/component) — label ตรงกับที่เห็นในโปรแกรม. เวอร์ชันแอปแสดงที่มุมล่าง Sidebar (v{version}).
