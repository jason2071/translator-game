User Manual

คู่มือ RPGMaker Translator

แอปแปลเกม RPG/VN ด้วยมือหรือ AI. ทุกเมนู เรียงตามลำดับตั้งค่า.
Flow: เปิดเกม → provider+key → context+glossary → Run → ตรวจ/แก้ → Export.

โครง

  • ยังไม่เปิดโปรเจกต์ → Import screen. เปิดแล้ว → Sidebar + (Update banner → TranslateBar → Grid).
  • 4 modal: Glossary, Glossary lint, AI providers & settings, Translation errors. ปิด: X / Esc / คลิกนอก.
  • Theme dark(default)/light.

Quit guard: ปิดหน้าต่างระหว่างแปล → เตือน. batch ที่เสร็จ save แล้ว, ที่เหลือหยุด.

เริ่มเร็ว 11 ขั้น

  1. Import → Choose game folder… → auto-detect (หรือ Recent = ข้าม detect, ใช้ภาษาเดิม)
  2. From(Auto) → To(Thai) → Open project (สกัดข้อความ)
  3. Settings → provider tab, Model, Base URL(local), Save key, ปรับค่า → Test connection
  4. เลือก Run provider (dropdown TranslateBar; ต้องมี key ยกเว้น Local)
  5. Glossary → panel บน: Era, glossary provider, AI draft, แก้ context
  6. glossary: Auto-suggest/AI suggest → Translate empty → Add selected
  7. scope (Sidebar: All files/ไฟล์) → Overwrite existing? → Run (Apply TM ก่อน = ฟรี)
  8. Grid: filter, แก้คำ (Ctrl/Cmd+Enter=save+ถัดไป), ตั้ง status
  9. Errors → Retry failed
  10. Lint → แก้คำไม่ตรง glossary
  11. Export → game (backup อัตโนมัติ; Embed Thai font = RPGMaker MV/MZ)

1. Import

  • Choose game folder… → picker. detect ล้ม → No supported game engine detected in this folder.
  • การ์ด detect: Engine, Data files, Data dir, warnings (ไม่บล็อก).
  • From: Auto/Japanese/English/Chinese/Korean (def Auto). To: Thai/English/Japanese/Chinese/Korean (def Thai).
  • Open project → สกัดข้อความ. Recent: badge + source→target + progress + X ลบ + Clear. เปิด Recent = ข้าม detect/ภาษา.

2. TranslateBar

  • ซ้าย: สลับภาษา; Run provider = Local(Ollama/LM Studio), OpenAI, Claude, Gemini, OpenRouter.
  • ขวา: Target: {file} (scope, เปลี่ยนที่ Sidebar); Overwrite existing; Run↔Cancel; Re-translate matches (N) (มี search); Retry failed (N); Errors (N).
  • ล่าง: progress Run+Glossary; สรุป Done — N translated, N reused, M failed; error ระดับคำสั่งแดง.
  • Run กับ Glossary แชร์ engine → ทำทีละงาน, อันที่สอง queue.

3. Sidebar

  • Dashboard: วง % = (total−untranslated)/total; {n} done(Translated+Reviewed), {n} todo, {n} failed.
  • Status filter: All, Todo, Failed, Draft, Translated, Reviewed, Locked.
  • Files filter: All files + ต่อไฟล์ → ตั้ง grid + Run scope.
  • ปุ่ม: Apply TM (เติมจาก TM+ซ้ำ), Glossary, Lint, Settings, Export → game, Embed Thai font (MV/MZ only, def on เมื่อ target=Thai), Close project.

4. Grid

  • search Search source / translation… (Enter); Untranslated only; N shown; ว่าง → Reset filters.
  • คอลัมน์: Source·context | Translation | Status.
  • Source: text + file + speaker + kind (Dialogue/Choice/Message…).
  • Translation: textarea, blur=save. Ctrl/Cmd+Enter=save+ถัดไป, Enter=บรรทัดใหม่. แก้ครั้งแรก Untranslated/Failed → Draft.
  • เตือน: ⚠ codes differ (inline code ไม่ตรง, engine-aware), ⚠ line too long (kind กล่อง + width guard).
  • ปุ่ม re-translate ต่อแถว (ใช้ Run provider, ทับ). Status dropdown: 6 ค่า.

5. Settings (AI providers & settings)

ตั้งแต่ละ provider อิสระ — ไม่เลือกว่าใครใช้ Run/glossary (Run→TranslateBar, glossary→context panel).

  • tabs: Local, OpenAI, Claude, Gemini, OpenRouter.
  • ต่อ provider: Model (+Refresh → — pick one of N installed —), Base URL (Local def http://localhost:11434/v1), Temperature (0–2, def 0.3), API key (Save/Clear, keychain; ทุกตัวยกเว้น Local).

  • Shared: Target tone(def casual), Batch size(1–200, def 40), Rate limit (req/min, 0=off)(def 0), Message width guard (chars, 0=off)(0–120, def 46), Thinking / reasoning(def off), Extra prompt (all projects).
  • Test connection → ✓/✗. (maxTokens มีใน data model แต่ไม่มีใน UI.)

6. Game context + Era

บนสุดของ Glossary modal, per-project.

  • Game context (this project) textarea (lore/ยุค/ตัวละคร/โทน) → ป้อน AI ทุก Run, save อัตโนมัติ.
  • AI draft = ร่างจากข้อความในเกม. Glossary provider dropdown (แยกจาก Run provider).
  • Era: none "", ancient (ข้า/เจ้า), medieval, wuxia, samurai, modern (ฉัน/คุณ), scifi. seed สรรพนาม/register ทุก Run. โมเดลอ่อนอาจไม่ตรง → Overwrite+Run ใหม่ / โมเดลแรงกว่า / แก้เอง.

7. Glossary

  • Auto-suggest from game (heuristic), AI suggest (glossary provider ขุด proper noun).
  • มี candidate: N candidates · N filled; Translate empty (N), Re-translate all (N), Add selected, Cancel.
  • แถว: checkbox + term (kind ×count) + translation + ⚠/✓ + retry. เก็บต่อเกม.
  • เพิ่มเอง: Source term, Translation, Note, Aa(case-sensitive), Add. ตาราง Term/Translation/Note/ลบ (แก้ inline).

8. Lint (Glossary lint)

  • รันอัตโนมัติ. สะอาด → ✓ No glossary inconsistencies found.
  • มี → N translation(s) miss their glossary wording. + {file} — term {X} should map to {Y} (คลิก→ไปไฟล์).

9. Errors (Translation errors)

  • เปิดจาก Errors (N). ไม่มี → ✓ No translation errors.
  • มี → list: source + file + เหตุผล. คลิก → filter failed ของไฟล์นั้น. retry ที่ Retry failed (N).

10. Export → game

  • backup เสมอ. Embed Thai font เฉพาะ MV/MZ. สำเร็จ → note ตาม engine (Ren'Py = tl/<lang>/) หรือ Exported N units → M files.
  • ไฟล์เกม read-only จนกว่า Export; state อยู่ใน <game>\.rpgtl\. Export idempotent (snapshot ต้นฉบับ).

Status / คีย์ลัด

Untranslated Failed Draft Translated Reviewed Locked
  • Untranslated=todo · Draft=แก้แล้วยังไม่ยืนยัน · Translated=AI เสร็จ · Reviewed=คนยืนยัน · Locked=ล็อก.
  • Keys: Ctrl/Cmd+Enter=save+ถัดไป; Enter(textarea)=บรรทัดใหม่; Enter(search)=ค้น; Esc=ปิด modal.

Dependency

  • เปิดโปรเจกต์ก่อน (ยกเว้น Settings). Run ต้องมี API key + Run provider (Local ไม่ต้อง).
  • glossary/context AI ใช้ glossary provider (แยกจาก Run). Run+Glossary ทำทีละงาน (queue).
  • Embed Thai font = MV/MZ only. Overwrite off = ข้ามที่แปลแล้ว. Apply TM ก่อน Run = ฟรี. Export backup เสมอ.

รันนานช้าลง? ส่วนใหญ่มาจาก provider (Ollama GPU ร้อน / API rate-limit) — นอกการควบคุมของแอป.
